1D Schrodinger Solver Web App

This guide is for a new user opening the solver for the first time. It explains what the app actually computes, which parameter limits are enforced by the implementation, which settings are only recommendations, and why some plotted states appear as accepted while others are marked as diagnostic or rejected.

Python 3.11+ Streamlit UI Dimensionless units Dirichlet boundaries only Finite-difference eigensolver

What This App Solves

The app computes low-lying stationary eigenstates of the one-dimensional time-independent Schrodinger equation using dimensionless units with hbar = 1 and m = 1. It is built for bounded-domain experiments, not for scattering, continuum physics, or time evolution.

What You See

The main plot overlays the sampled potential V(x) and each numerical eigenfunction shifted upward by its energy. A separate energy plot shows horizontal energy lines. Accepted states are solid; rejected states are dashed red diagnostics.

What "Rejected" Means

Rejected does not mean "no solution was computed." It means the post-solve validator judged that the state is low confidence under the app's current thresholds. The curves are still plotted so you can inspect them visually.

Core Mathematics

The continuous problem solved by the app is:

- 1 2 d2psi dx2 + V(x) psi(x) = E psi(x)

The implementation assumes a finite box with Dirichlet boundary conditions:

psi(xmin)=0 , psi(xmax)=0

The spatial grid is uniform:

xi = xmin + iDeltax

The second derivative is approximated with a centered finite difference:

d2psi dx2 ≈ psii+1 - 2psii + psii-1 Deltax2

This produces a tridiagonal Hamiltonian on the interior grid:

Hii = 1Deltax2 + V(xi) Hi,i±1 = - 12Deltax2

The eigenvectors are expanded back to the full grid, with boundary values fixed to zero, and normalized numerically:

∫xmin xmax |psi(x)|2 dx =1
The solver calls scipy.linalg.eigh_tridiagonal, which is efficient and appropriate for the symmetric tridiagonal matrix produced by this discretization.

Potential Input Modes

1. Template mode

Built-in presets: Infinite Square Well, Harmonic Oscillator, Finite Square Well, and Barrier / Step.

2. Analytic mode

Enter a scalar expression for V(x). The parser accepts only a restricted safe subset of Python syntax.

3. Piecewise mode

Define contiguous constant-valued segments that must exactly cover the selected domain.

Analytic Parser Rules

Allowed names: x, pi, e.

Allowed functions: sin, cos, tan, sinh, cosh, exp, sqrt, abs, log.

Allowed operators: +, -, *, /, **.

Unsupported symbols, keyword arguments, imports, indexing, attribute access, and arbitrary code execution are rejected before solve.

Validation Logic and Why States Get Rejected

After a solve, the app automatically recomputes the same problem on a refined grid and compares each state. A state is accepted only if both checks pass:

|psi1| ≤1e-4 , |psiN-2| ≤1e-4 |Ecoarse - Erefined| ≤1e-3

In code, the boundary test uses the first and last interior amplitudes, not the exact endpoints. That is conservative. It is why hard-wall and weakly confined states can be numerically reasonable yet still show up as rejected.

A rejected state is still plotted. Export buttons are disabled only when no state passes validation.

Practical Starting Bounds for New Users

The table below separates "allowed by the UI" from "usually a good first run." These are not physics laws. They are operational recommendations based on the actual validation behavior in this implementation.

Potential Recommended first domain Recommended grid Recommended modes Expected behavior
Harmonic oscillator [-5, 5] or [-8, 8] 401 to 801 1 to 4 Best starting point. These settings consistently pass validation for low states.
Infinite square well Match the full domain to the well exactly 801+ 1 to 3 Good for benchmark comparison, but often marked rejected by the boundary threshold.
Finite square well At least several well-widths wider than the well 801+ 1 to 2 Bound states can look correct, but current drift threshold commonly rejects them.
Barrier / step Use a wide box, for example [-8, 8] or wider 1201+ 1 to 2 Frequently rejected because box-confined states retain noticeable edge amplitude.
Analytic custom potential Start with a confining potential centered away from boundaries 401 to 801 1 to 4 Use smooth potentials first, such as 0.5 * x**2.
Piecewise custom potential Prefer box-like bound wells over open scattering profiles 801+ 1 to 2 Works best when the physical state decays well before the domain edges.
Recommended first run for a new user: Harmonic Oscillator, domain [-5, 5], grid 401, modes 4, k = 1, center 0, offset 0.
Not every physically meaningful setup will be accepted by the validator. In the current version, the acceptance logic favors strongly confined states and can be too strict for infinite wells, finite wells, and barrier-like problems.

Template Equations

Harmonic oscillator

V(x) = 12 k (x-center)2 +offset

Infinite well

V = 0 inside the well and wall_height outside.

Finite well

V = outside - depth inside the well and outside outside.

Barrier

V = barrier_height inside the barrier interval and baseline elsewhere.

Analytic Benchmarks

Infinite square well energies

En = n2pi2 2L2

Harmonic oscillator energies

En = omega(n+12) +offset , omega=k

The app can show an analytic comparison only for harmonic oscillator and for infinite square well when the well exactly spans the selected domain.

How to Read the Plots

Main plot: black curve is V(x); each colored curve is psi(x) + E.

Energy plot: one horizontal line per state.

Benchmark plot: solid line is numerical psi(x); dashed line is analytic psi(x).

The shifted eigenfunction view is visual only. It helps you see shape and energy on a shared axis, but it is not a direct probability-density plot.

Operating Procedure for a New User

  1. Launch the app with schrodinger-solver-web or streamlit run src/schrodinger_solver_web/app.py.
  2. Start in Template mode and choose Harmonic Oscillator.
  3. Keep the first run simple: domain [-5, 5], grid 401, modes 4.
  4. Click Solve.
  5. Read the solve summary first, then inspect the solution and energy plots.
  6. Open Validation details only if states were rejected or if you need drift and edge-amplitude diagnostics.
  7. Use benchmark comparison for supported templates when you want an analytic cross-check.
  8. Export CSV or PNG only after at least one state is accepted.

What the CSV and PNG Exports Contain

CSV

The CSV begins with metadata rows describing the input mode, selected domain, grid points, requested modes, unit convention, and any template parameters or piecewise segments. The data table then includes:

  • x and V
  • n_k_psi for each accepted state
  • n_k_probability for each accepted state
  • Accepted and rejected state labels as metadata columns

PNG

The PNG export is the static image version of the main Plotly solution figure. It captures the potential and the shifted eigenfunction curves exactly as displayed in the app.

Known Limitations

  • No time-dependent Schrodinger equation support.
  • No alternative boundary conditions beyond the fixed zero-value box.
  • No continuum-state or scattering-state treatment.
  • No persistence layer for saved sessions or custom presets.
  • Validator thresholds are intentionally strict and can reject states that are visually reasonable.
  • Benchmark comparison is intentionally narrow and only covers two analytic families.
